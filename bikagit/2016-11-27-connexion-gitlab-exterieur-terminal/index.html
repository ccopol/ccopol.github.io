Bikagit hors de l'université ! (méthode avec le terminal)

Wed, Jan 27, 2016

L’utilisation de bikagit au sein de l’université de pose aucun problème1 mais hors de l’université cela vous demandera quelques adaptations. Dans la suite, nous verrons comment vous pourrez accéder à vos dépôt bikagit à l’extérieur.

Pré-requis

Accès

La création d’un compte permettant l’accès au service bikagit n’est pas un processus automatique, assurez-vous donc d’avoir un compte.

Nous supposons que vous avez un compte sur le bikagit de l’université mais aussi sur le sas. Si l’un de ces pré-requis n’est pas satisfait, veuillez vous rapprocher de votre administrateur de bikagit et de sasc3i.

Convention

Tout d’abord, prenons quelques conventions. Dans le reste du document vous devrez remplacer user et repository par, respectivement votre username (certainement votre login sur bikagit) et par le dossier auquel vous voulez avoir accès.

Supposons aussi que le dossier local (sur votre ordinateur) contenant votre travail se trouve dans /home/user/repository

Résumé

Il vous faut :

  • un compte sur
    • bikagit
    • sasc3i
  • terminal

Ouverture d’un accès

Pour ouvrir un accès au sasc3i, ouvrez un terminal et garder le ouvert pendant toute la séance de travail sur git. Attention ! Fermez le terminal c’est fermer la porte d’accès. Tapez dans le terminal

~$ ssh -L 2022:lic3i.univ-ag.fr:22 user@sasc3i.univ-ag.fr

Rajout du dépôt distant

Se rendre (avec un autre terminal) dans le dossier contenant votre travail.

~$ cd /home/user/repository

Lister les dépôts

Vous pouvez afficher la liste de dépôts distant associés à ce répertoire

~$ git remote -v

Vous devriez avoir aux minimum ceci

origin    git@lic3i.univ-ag.fr:user/repository.git (fetch)
origin    git@lic3i.univ-ag.fr:user/repository.git (push)

« origin » est le surnom de du dépôt git@lic3i.univ-ag.fr:user/repository.git.

Ajouter un dépôt

Il faut rajouter le dépôt distant ssh://git@localhost:2022/user/repository.git que nous pouvons surnommer « ext ».

~$ git add ext ssh://git@localhost:2022/user/repository.git

Si vous lister à nouveau les dépôts, vous avez quelque chose comme ceci :

ext     ssh://git@localhost:2022/user/repository.git (fetch)
ext     ssh://git@localhost:2022/user/repository.git (push)
origin  git@lic3i.univ-ag.fr:user/repository.git (fetch)
origin  git@lic3i.univ-ag.fr:user/repository.git (push)

Push it

~$ git push ext master

Pull it

~$ git pull ext master
